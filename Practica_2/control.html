10. ¿Cómo funcionan las sentencias condicionales if, else y switch?

Estas sentencias controlan el flujo de la ejecución. Las sentencias if y else se usan para ejecutar sentencias de codigo en base a la condicion boleana: if si es verdadero, else es una alterativa y else if contiene más condiciones. 

            let numero = 10;

            if (numero >= 10){
                console.log("True");
            } else {
                console.log("False");
            }
Mientras que el switch se usa para la seleccion entre varias opciones. 
            opcion = 1;

            switch(opcion){
                case 1: 
                    console.log("Se ingreso 1");
                break;
                case 2: 
                    console.log("Se ingreso 2");
                break;
                defautl: 
                    console.log("Ingreso algo bien raro");
                break;
            }

11. ¿Cuáles son las diferencias entre los bucles for, while y do...while?

For

Es una estructura definida con iteraciones, es ideal cuando se tiene un número definido de iteraciones que se van a realizar, como en el recorrido de un array. 

            for (int i = 0; i <= 10; i++) {
                 // Contenido
            }

While

El ciclo while es un control de flujo que repetira que se declara con una condición, este ciclo se va a estar iterando hasta que la condicion se rompa. Es ideal cuando no sabemos que tantas veces se va a repetir el codigo, si ninguna o muchas. 

            while (condicion) {
                 // Contenido
            }

Do... While

A diferencia del while en el do while tenemos definido que el contenido del ciclo se va a ejecutar al menos una vez sin importar la condición, después se evalua la condicion. 

            do {
                 // Contenido
            } while (condicion);

12. ¿Para qué sirven las palabras clave break y continue?

Las palabras clave de break y continuo se usan para controlar el flujo del programa. El break se sale completamente del blucle, mientras que continue omite el resto del codigo de la iteración actual pasansose a la siguiente.
